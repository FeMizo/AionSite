## 1. Fundamentos de un Sitio Web Efectivo

### Propósito Claro y Audiencia Definida

Antes de escribir una sola línea de código, debes responder:
- **¿Cuál es el objetivo principal?** (Ventas, leads, información, comunidad)
- **¿Quién es tu audiencia?** (Datos demográficos, necesidades, dolor points)
- **¿Cuál es tu propuesta de valor?** (Qué te diferencia de la competencia)

Un sitio web sin claridad es como un barco sin brújula. Navega, pero no llega a puerto.

### Arquitectura de Información Intuitiva

La estructura de tu sitio debe permitir que los visitantes encuentren lo que buscan en máximo 3 clics. Una buena arquitectura incluye:

- **Menú principal claro y conciso** (no más de 6-7 items)
- **Categorías lógicas y jerárquicas**
- **Búsqueda interna efectiva**
- **Breadcrumbs para orientación**

## 2. Diseño que Convierte

### Principios de Diseño Conversión-Centric

El diseño no es solo estética. Es psicología aplicada. Elementos clave:

#### Call-to-Actions (CTAs) Efectivos
- **Contraste visual:** Los botones deben destacar del fondo
- **Texto claro y directo:** "Obtener Demo" en lugar de "Enviar"
- **Posicionamiento estratégico:** Visible sin necesidad de scroll cuando es posible
- **Múltiples CTAs:** A lo largo del viaje del usuario, no solo al final

#### Uso del Espacio en Blanco
El espacio en blanco no es "vacío". Es respiro. Permite que el ojo descanse y dirige la atención hacia elementos importantes.

#### Tipografía Clara
- Tamaño mínimo de 16px para body text
- Máximo 2-3 tipografías por sitio
- Contraste suficiente para accesibilidad (WCAG AA mínimo)

#### Paleta de Colores Cohesiva
Los colores evocan emociones y comportamientos. Elige una paleta que:
- Represente tu marca
- Tenga suficiente contraste
- Sea coherente en todo el sitio

### Responsive Design: Obligatorio, no Opcional

En 2026, más del 60% del tráfico proviene de móviles. Tu sitio DEBE funcionar perfectamente en:
- Teléfonos (320px - 480px)
- Tablets (600px - 1024px)
- Desktops (1200px+)

## 3. Velocidad: El Factor Oculto de Conversión

### Por Qué la Velocidad Importa

- **Cada segundo de demora = 7% menos conversiones** (estudios de Google)
- Google usa Core Web Vitals como factor de ranking
- Los usuarios abandonan sitios lentos al instante

### Optimizaciones Clave

#### Compresión de Imágenes
- Usa formatos modernos: WebP, AVIF
- Comprime sin perder calidad visual
- Implementa lazy loading

#### Minificación de Assets
- CSS y JavaScript minificados
- Eliminación de código no usado
- Bundling inteligente

#### Caché y CDN
- Implementa caché del navegador
- Usa CDN global para servir contenido desde ubicación cercana al usuario
- Caché de aplicación para navegación más rápida

#### Eliminación de Bloqueadores de Rendering
- JavaScript deferido o async
- CSS crítico inline
- Fuentes web optimizadas

## 4. SEO: Fundamentos para Posicionamiento

### SEO On-Page

#### Investigación de Palabras Clave
- Identifica términos con intención de búsqueda relevante
- Analiza competencia
- Busca keywords long-tail con menor competencia

#### Optimización de Meta Tags
```
<title>Máx. 60 caracteres - Palabra clave principal</title>
<meta name="description" content="120-160 caracteres. Atractivo y con CTA.">
<meta name="robots" content="index, follow">
```

#### Estructura de Encabezados
```
H1: Una por página, con palabra clave
H2: Subsecciones temáticas
H3: Detalles y subtemas
```

#### URLs Amigables
- `/creacion-sitios-web-optimizados` ✅
- `/article?id=123&cat=web` ❌

### SEO Técnico

- **XML Sitemap:** Ayuda a Google a descubrir contenido
- **Robots.txt:** Controla qué se indexa
- **Structured Data (Schema.org):** Ayuda a motores de búsqueda a entender contenido
- **HTTPS obligatorio:** Factor de ranking y confianza

### SEO Off-Page

- **Backlinks de calidad:** Links desde sitios relevantes y autorizados
- **Menciones de marca:** Referencias sin link también cuentan
- **Social signals:** Actividad en redes sociales indirectamente afecta ranking

## 5. Seguridad y Confianza

### Certificado SSL/TLS
Todo sitio debe tener HTTPS. Es:
- Requisito de Google
- Esencial para datos de usuarios
- Visible en la barra de dirección

### GDPR y Privacy
- Política de privacidad clara
- Consentimiento para cookies
- Manejo seguro de datos

### Seguridad de Código
- Actualizaciones regulares de dependencias
- Validación de inputs
- Protección contra SQL injection y XSS

## 7. Análisis y Mejora Continua

### Métricas Clave (KPIs)

#### Performance
- **Core Web Vitals:**
  - LCP (Largest Contentful Paint): < 2.5s
  - FID (First Input Delay): < 100ms
  - CLS (Cumulative Layout Shift): < 0.1

#### Conversión
- Tasa de conversión (%)
- Costo por conversión
- Valor de vida del cliente (LTV)

#### SEO
- Posición promedio en SERP
- Clics orgánicos
- Impresiones en búsqueda

### Herramientas de Análisis

- **Google Analytics 4:** Comportamiento de usuarios
- **Google Search Console:** Visibilidad en búsqueda
- **PageSpeed Insights:** Rendimiento
- **Hotjar/Clarity:** Comportamiento y heatmaps

## 8. Errores Comunes a Evitar

❌ **Priorizar estética sobre funcionalidad**  
✅ Diseño hermoso que convierte

❌ **Ignorar móviles**  
✅ Mobile-first approach

❌ **Cargar demasiado contenido sin estructura**  
✅ Contenido bien jerarquizado y escaneable

❌ **Usar stock photos genéricos**  
✅ Imágenes auténticas o ilustraciones originales

❌ **CTAs débiles o escondidos**  
✅ CTAs claros, contrastados y estratégicamente posicionados

❌ **No medir resultados**  
✅ Analytics desde el día uno
